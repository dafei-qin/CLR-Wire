### 2. 从files.txt读取文件列表

**优势：**
- 避免在云存储上执行慢速的find/ls操作
- 文件列表预先生成，读取速度快
- 支持断点续传（检查*_processing.log）

**实现：**
```bash
# 读取files.txt
while IFS= read -r rel_path; do
    full_path="${INPUT_BASE}/${tens}/${two_digit}/${rel_path}"
    STEP_FILES+=("$full_path")
done < "$files_txt"
```

# ABC Dataset Processing Pipeline

## 项目概述
对 ABC CAD 数据集的 STEP 文件进行批量点云采样处理。

## 数据
- 源文件：`/data/logan/abc/abc_XXXX_step_v00.7z`，共 100 个（0000~0099）
- 每个 7z 包含约 10000 个 STEP 文件，子目录结构如 `00009999/00009999_xxx.step`

## 核心脚本
- **end2end 脚本**：`/data/logan/abc/end2end_step_to_pc.sh`
  - 流程：解压 7z → 批处理 STEP → 清理 temp → 下一个
  - 用法：`bash end2end_step_to_pc.sh 4 5 6 7`（处理 004x~007x）
  - 参数 group 表示十位数，`6` = abc_0060~0069，`0` = abc_0000~0009
  - 批处理参数（并发数、面数等）在脚本顶部直接修改
  - 已处理的 chunk（输出目录非空）自动跳过
- **参考批处理脚本**：`/root/CAD/CLR-Wire/scripts/batch_step_to_pc_v2_SHT_disk2.sh`
- **Python 处理脚本**：`/root/CAD/CLR-Wire/src/tools/sample_step_to_pc_v2_debug.py`

## 已知坑
- bash 内置变量 `GROUPS` 是只读的（存储用户 Unix 组），不能用作自定义数组名，已改用 `GROUP_LIST`

## 路径约定
| 用途 | 路径 |
|------|------|
| 7z 源文件 | `/data/logan/abc/` |
| 解压临时目录 | `/root/CAD/data/temp` |
| 输出目录 | `/data/ssd/CAD/data/abc_step_pc_v2/XXXX/` |
| 日志目录 | `/data/ssd/CAD/data/abc_step_pc_v2/logs/end2end/` |
| CLR-Wire 项目 | `/root/CAD/CLR-Wire/` |

## 默认批处理参数
- `MAX_PARALLEL=32`，`N_POINTS=40960`，`DEFLECTION=0.005`
- `MIN_FACES=20`，`MAX_FACES=500`，`DEBUG=false`
